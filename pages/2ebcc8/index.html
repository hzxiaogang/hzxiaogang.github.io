防抖和节流

# 函数防抖&节流

# 函数防抖

函数防抖(debounce)：当持续事件发生时，在一定时间内没有再触发该事件，事件处理函数才会执行一次。如果设定的时间来到之前，又一次触发了事件，就重新开始延时。 如上图，持续触发scroll事件，并不执行 handle函数，当1000毫秒内没有再触发scroll事件时，才会延时触发scroll事件。

# 非立即执行版本

非立即执行版的意思是触发事件后函数不会立即执行，而是在 n 秒后执行，如果在 n 秒内又触发了事件，则会重新计算函数执行时间。

const debounce = (fn, wait, ...args) => {
  let timeout
  let _this = this
  return function () {
    if (timeout) {
      clearTimeout(timeout)
    }
    timeout = setTimeout(() => {
      fn.apply(_this, args)
    }, wait)
  }
}

# 立即执行版本

立即执行版的意思是触发事件后函数会立即执行，然后 n 秒内不触发事件才能继续执行函数的效果

const debounce = (fn, wait, ...args) => {
  let timeout
  let _this = this
  return function () {
    if (timeout) {
      clearTimeout(timeout)
    }
    let callNow = !timeout
    timeout = setTimeout(() => {
      timeout = null
    }, wait)
    if (callNow) {
      fn.apply(_this, args)
    }
  }
}

# 函数节流

函数节流（Throttle）: 所谓节流，就是指连续触发事件但是在 n 秒中只执行一次函数。节流会稀释函数的执行频率。

使用时间戳

const throttle = (fn, wait, ...args) => {
  let pre = 0
  let context = this
  return function () {
    let now = new Date().getTime()
    if (now - pre >= wait) {
      fn.apply(context, args)
      pre = new Date().getTime
    }
  }
}

使用定时器

const throttle = (fn, wait, ...args) => {
  let timeout
  let context = this
  return function () {
    if (!timeout) {
      timeout = setTimeout(() => {
        if (timeout) {
          fn.apply(context, args)
        }
        timeout = null
      }, wait)
    }
  }
}
