函数

# 简介

函数是JavaScript应用程序的基础。它帮助你实现抽象层，模拟类，信息隐藏和模块。在TypeScript里，虽然我们已经支持类，命名空间和模块，但是函数任然是主要的定义行为的地方。TypeScript为函数添加了额外的功能，让我们可以更容易的使用。和JavaScript一样，TypeScript函数可以创建有名字的函数和匿名函数，你可以任意选择合适应用程序的方式，不论是定义一些列API函数还是只使用一次的函数。

// Named function (有名字的函数) (声明的方式创建函数)
function add(x,y) {
  return x + y;
}

// Anonymous function (匿名函数)  (通过函数表达式创建函数)
let myAdd = function(x,y){return x + y;}

# 函数类型

# 书写完整函数类型

函数的类型由参数类型和返回值类型组成，这两部分都是必须的。我们以参数列表的形式写出参数类型，为每个参数指定一个名字和类型，这个名字只是为了增加可读性，可以随意写，只要参数类型是匹配的，那么就认为它是有效的参数类型。返回值类型的写法因创建函数的方式不同而不同，以声明的方式创建函数，其返回值类型写法是 :加上数据类型。以函数表达式的形式创建函数，其返回值类型的写法是 =>加上数据类型。下面的两个例子书写了完整的函数类型。

// 声明的方式创建函数
function add(x: number, y: number): number {
    return x + y;
}
// 函数表达式的方式创建函数
let myAdd: (x: number, y: number) => number =
    function(x: number, y: number): number { return x + y; };

# 推断类型

创建函数的时候，当你在赋值语句的一边指定了类型，但是另一边没有类型的话，TypeScript编译器会自动识别出类型:

// myAdd1 has the full function type
let myAdd1 = function(x: number, y: number): number { return x + y; };

// The parameters `x` and `y` have the type number
let myAdd2: (baseValue: number, increment: number) => number =
    function(x, y) { return x + y; };

# 使用接口创建函数类型

上面的方式书写完整的还是类型太过于复杂，我们可以通过使用interface(接口)的方式创建一个函数类型，方式如下:

interface funType{
  (x:number,y:number):number
}
let add : funType = function(x:number,y:number):number{
  return x + y;
}

# 函数的参数

TypeScript里的每个参数都是必须的，这不是指不能传递undefined和null，而是说编译器检查用户是否给每个参数都传了值。简单的说就是:传递给函数的参数的个数必须要与函数期望的参数个数一致。根据功能不同，函数的参数有可选参数，默认参数，剩余参数这三种。

# 可选参数

JavaScript里，每个参数都是可选的，可传可不传。没传参的时候，它的值就是undefined。在TypeScript里我们可以在参数名旁使用?实现可选参数功能。可选参数有2个特点：

  • 可选参数必须跟在必需参数后面(当有必须参数的时候)
  • 当一个可选参数没有传值的时候，它的值为 undefined
function buildName (firstName:string,lastName?:string){
  if(lastName)
    return firstName + " " +　lastName;
  }else{
    return firstName;
  }
}
let result1 = buildName('Bob'); // "Bob"
let result2 = buildName('Bog','Adams'); // "Bog Adams"
let result3 = buildName('Bog','Adams','Sr.') // error,  too many parameters

# 默认的参数

在TypeScript里我们可以为参数提供一个默认值，当用户没有传递这个参数或传递的值是undefined。默认参数有两个特点;

  • 默认参数不指定参数类型时，其类型就是默认参数值的类型。
  • 默认参数放在必需参数的前，后都可以。如果默认参数出现在必需参数前面，用户必须明确的传入undefined值来获取默认值。
function buildName(firstName: string, lastName:string = "Smith") {
    return firstName + " " + lastName;
}
或
//默认参数不指定参数类型时，其类型就是默认参数值的类型。 所以lastName的参数类型可以不写
function buildName(firstName: string, lastName = "Smith") {
    return firstName + " " + lastName;
}
let result1 = buildName("Bob");                  // "Bob Smith"
let result2 = buildName("Bob", undefined);       // "Bob Smith"
let result3 = buildName("Bob", "Adams", "Sr.");  // error, too many parameters
let result4 = buildName("Bob", "Adams");         // "Bob Smith"

# 剩余参数

必要参数，默认参数和可选参数有一个共同点： 它们表示某一个参数。有时我们想操作多个参数，或者我们并不知道会有多少个参数传递进来。在JavaScript里我们可以使用 arguments，rest参数来访问所有传入的参数。在TypeScript 里，你可以把所有参数搜集到一个变量里。其实我觉得它就是ES6里面的rest参数，只是写法不同而已。

function buildName(firstName : string , ...restOfName:string[]){
  return firstName + " " + restOfName.join(" ");
}
let employeeName = buildName("Joseph", "Samuel", "Lucas", "MacKinzie"); // "Joseph Samuel Lucas MacKinzie"

# this参数

JavaScript里，this的值在函数被调用的时候才会指定。这是个既强大又灵活的地方，但是你需要话点时间弄清楚函数调用的上下文，这不是一件很简单的事情，尤其是在返回一个函数或将函数动作参数传递的时候。在TypeScript里我们可以在函数里指定this参数的类型，这样在函数编译的时候，就已经确定好了调用的上下文。this参数是一个假的参数，它出现在参数列表的最前面：

// 必须且默认是 void，否则报错
function f(this: void) {
    // make sure `this` is unusable in this standalone function
}

现在TypeScript知道createCardPicker期望在某个Deck对象上调用。 也就是说 this是Deck类型的。

interface Card {
    suit: string;
    card: number;
}
interface Deck {
    suits: string[];
    cards: number[];
    createCardPicker(this: Deck): () => Card;
}
let deck: Deck = {
    suits: ["hearts", "spades", "clubs", "diamonds"],
    cards: Array(52),
    // NOTE: The function now explicitly specifies that its callee must be of type Deck
    createCardPicker: function(this: Deck) {
        return () => {
            let pickedCard = Math.floor(Math.random() * 52);
            let pickedSuit = Math.floor(pickedCard / 13);

            return {suit: this.suits[pickedSuit], card: pickedCard % 13};
        }
    }
}

let cardPicker = deck.createCardPicker();
let pickedCard = cardPicker();

alert("card: " + pickedCard.card + " of " + pickedCard.suit);

# 重载

方法重载就是 ： 方法名称相同，参数列表却不相同(个数或类型不同)，当调用函数的时候，会根据传入的参数不同去调用不同的函数。重载方法返回值可以相同也可以不相同。方法重载在静态类型语言中很常见(Java,C#,C++)，JavaScript作为动态语言，是没有重载这一说的，一是它的参数没有类型，二是对参数个数也没有检查。TypeScript也不是严格意义上的重载，值是利用了重载的思想。TypeScript实现重载的方式是：为同一个函数提供多个函数类型定义来进行函数重载。

function reverse(x: number): number;
function reverse(x: string): string;
function reverse(x): any {
    if (typeof x === 'number') {
        return Number(x.toString().split('').reverse().join(''));
    } else if (typeof x === 'string') {
        return x.split('').reverse().join('');
    }
}

console.log(reverse(123))
console.log(reverse('abc'))

上例中，我们定义了多个reverse函数，前面几个都是函数定义，最后一次是函数实现，这样一来形成一个重载列表。编译器会根据这个重载列表去处理函数调用。当调用reverse函数的时候，编译器会查找重载列表。尝试使用第一个重载定义。如果匹配的话就使用这个。因此在定义重载的时候，一定要把最精确的定义放在最前面。本例子中function reverse(x): any并不是重载列表的一部分，因此这里只有两个重载。一个接收数字，一个接收字符串。以其它参数调用reverse会报错。
